                           ↓

## 系统内部处理流程

```
用户点击"截图求助"
    ↓
Python: 截取右侧面板
    ↓
Python: 上传截图到Dify (/v1/files/upload)
    → 获得file_id
    ↓
Python: 调用本地匹配接口 (/api/match-fragment)
    ↓
    ├─ OCR提取关键词
    │  └─ 调用Dify Chat API
    │     └─ Prompt: "提取界面上的标题、按钮、标签文字"
    │     └─ 返回: ["社保信息", "是否申请社保", "咨询电话12333"]
    │
    ├─ 本地匹配片段
    │  └─ 加载fragments_structured.json
    │  └─ 三层混合匹配:
    │     1. 完全匹配 ≥2个关键词
    │     2. 1个稀有关键词（频率≤2）
    │     3. 部分匹配 ≥2个关键词
    │  └─ 返回: fragment + 匹配方法 + 分数
    │
    └─ 返回匹配结果给前端
       {
         "fragment": {...},
         "match_method": "完全匹配3个关键词",
         "match_score": 45.0
       }
    ↓
前端 (chat.html): 构建上下文
    ├─ 步骤: 第11步 · 填写申请信息
    ├─ 标题: 社保信息填写
    ├─ 关键词: 社保信息, 是否申请社保
    ├─ 注意事项: 社保可先不申请...
    └─ 操作说明: 原始操作说明...
    ↓
前端: 调用Dify Chat API (/v1/chat-messages)
    ├─ query = "请根据截图..." + 上下文信息
    ├─ files = [截图]
    └─ streaming = true
    ↓
Dify: 基于上下文生成回答
    ├─ 识别截图内容
    ├─ 结合片段信息
    ├─ 提取操作步骤
    ├─ 添加注意事项
    └─ 警告常见错误
    ↓
前端: 流式显示AI回答
    └─ 用户看到完整的填写指导
```

## 关键时间节点

| 阶段 | 耗时 | 说明 |
|------|------|------|
| 截图 | <0.5s | Python本地截图 |
| 上传 | 1-2s | 上传到Dify服务器 |
| OCR提取 | 2-5s | Dify Chat API处理 |
| 本地匹配 | <0.01s | 本地计算，极快 |
| AI回答 | 3-8s | 生成详细指导 |
| **总计** | **6-15s** | 用户等待时间 |

## 匹配策略详解

### 三层混合匹配（优先级从高到低）

#### Level 1: 完全匹配（≥2个关键词）
```python
OCR关键词: ["社保信息", "是否申请社保", "咨询电话12333"]
片段keywords: ["社保信息", "咨询电话12333", "是否申请社保"]
交集: ["社保信息", "是否申请社保", "咨询电话12333"]
交集大小: 3 ≥ 2 ✅
匹配成功，分数 = 3×10 + 稀有加权
```

#### Level 2: 稀有关键词（1个）
```python
OCR关键词: ["多证合一"]
片段keywords: ["多证合一", "税务信息", ...]
关键词频率: "多证合一"只出现1次（稀有关键词）
匹配成功，分数 = 10 + 5（稀有奖励）
```

#### Level 3: 部分匹配（≥2个）
```python
OCR关键词: ["社保", "是否申请"]
片段keywords: ["社保信息", "是否申请社保"]
包含关系:
  - "社保" 包含于 "社保信息" ✅
  - "是否申请" 包含于 "是否申请社保" ✅
部分匹配数: 2 ≥ 2 ✅
匹配成功，分数 = 2×5
```

### 匹配失败场景

| 场景 | 原因 | 解决方案 |
|------|------|---------|
| OCR提取失败 | 截图模糊/文字过小 | 用户重新截图，确保文字清晰 |
| 关键词不匹配 | keywords覆盖不全 | 人工补充该场景的keywords |
| 匹配到错误片段 | 高频词误匹配 | 增加稀有关键词权重 |

3. AI回答:
   ✓ 这是第11步·填写申请信息
   ✓ 社保可先不申请，注册成功后可自行办理
   ✓ 建议银行开户成功后再开立社保
   ✓ 每个月1-12号申报和扣费

用户决策:
看到提示后选择"否"，跳过社保申请
```

## 故障排查流程

### 用户侧检查清单

```
问题：点击"截图求助"没反应

检查：
□ 应用是否正常启动（看到左右两个面板）
□ 右侧是否加载了网页（能看到表单）

## 最佳实践

### 用户侧

1. **截图时机**: 在表单步骤停留时截图，不要在滚动/加载中截图
2. **文字清晰**: 确保截图中的标题、按钮文字清晰可见
3. **逐步填写**: 每个步骤都可以截图求助，不要急躁

### 开发侧

1. **keywords维护**: 定期检查匹配覆盖率，补充新场景
2. **tips更新**: 根据用户反馈更新注意事项
3. **性能优化**: OCR提取耗时最长，考虑缓存机制
4. **错误日志**: 记录匹配失败案例，持续优化
